Option A Backend Token Exchange

Hero API exchanges the Keycloak access token (stored server-side) for a DIPS token using requested_issuer (Keycloak V1 token exchange). The SPA never sees the Keycloak token.

Option B Frontend Silent Authentication

The SPA performs a prompt=none authorization request directly against the external IdP (mock-oauth2-server / DFS), leveraging the SSO cookie from the initial federated login. The token is obtained entirely in the browser.
